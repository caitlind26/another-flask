AAA Testing



The Importance of Testing


There is no point in making code for functions if it does not have functionality. Testing the functionality of a code is critical in ensuring that the function does what it is intended to do. Creating simple unit tests to check each function is a good way of making sure that the code works for a certain test case. If the test case passes the pytest, then other applications of that code will work as well.

Testing with pytest


Running pytest in the terminal will allow you to see if the unit tests all pass. If the unit test has errors, then the pytest will tell you the exact line of the error and where the error is.

What is AAA Testing?


AAA testing is a simple pattern to structure unit tests. It focuses on testing specific behaviors in your code to make sure that it is functional. These tests are simple and ensures that the function does what it is intended to do.


  1. Arrange: This step requires you to sets up the test case by putting in targets. If the test requires an inputs, preparations, or logins, this is where you should specify that information.
  2. Act: This step covers the main item of testing. The action should be focused on the target behavior of the function.
  3. Assert: This step specifies the expected outcome to ensure that the proper result is obtained. The Assert step verifies that the desired response is elicited.

AAA Tutorial


Say suppose I wanted to create a test for dividing two numbers and getting the quotient.



def test_division_of_nums():
#Arrange value_1 = 6 value_2 = 3
#Act quotient = divide(value_1,value_2)
#Assert assert quotient == 2

  1. Arrange: This step specifies the values for the operation.
  2. Act: This step calls the division function and stores the answer in a variable called "quotient".
  3. Assert: This step verifies that the result of the division operation is correct.
