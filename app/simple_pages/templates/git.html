Git

Terminology

  • Branches: Branches are used to encapsulate changes in the development process. A branch represents the editing and committing process. The git branch command allows you to create, list, rename, and delete branches. Branches can be merged into the master branch so that the main code can reflect the changes you made.
  • Merge:The merge command allows you to merge branches from Git. Merging branches into the main code allows you to put all your changes into one main branch.
  • Commit: A commit captures the individual change to a file or set of files.
  • Tags:tags refer to specific points in Git history. Tags do not change like how a branch can after being created. Tags typically capture a point in history that is used for a marked version release
  • Repositories:A repository is a collection of files and it tracks all the changes in the files of your folders
