Docker

Containers & Images

A container is a unit of software that includes code and dependencies that the application needs to run on any computer. An image is an executable package of software that includes code, runtime, system tools, system libraries, and settings Container images become containers when the application is run. With Docker containers, the images become containers when they run on the Docker Engine. Containers separate the software and the environment to see that they work independently. Docker containers can be used on Linux, Windows, Cloud, and more.

Docker Commands

  • docker compose up -- build: builds docker image from Dockerfile
  • docker --version: shows Docker version information
  • docker run: creates a container layer over an image
  • docker stop: stops running a container
  • docker ps: displays list of running containers
  • docker commit: creates a new image from the container's changes
  • docker push: pushes an image/repo to a registry
  • docker pull: pulls an image/repo from a registry
  • docker rm: removes containers
